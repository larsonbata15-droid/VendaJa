Meta Pixel (Facebook & Instagram)

Liga o teu Pixel da Meta à tua loja para medires os resultados dos teus anúncios e criares públicos personalizados. Os eventos são enviados automaticamente — não precisas de tocar em código.

O ID do Pixel deve ter entre 15 e 16 dígitos.

Encontra-o no Gestor de Eventos da Meta → Fontes de dados → o teu Pixel. É um número com 15 ou 16 dígitos.

Como testar
1. Cola o ID do Pixel acima e guarda.
2. Instala a extensão Meta Pixel Helper no Chrome.
3. Abre a tua loja pública e confirma que o Pixel aparece a verde.
4. No Gestor de Eventos da Meta, usa "Testar eventos" e cola o link da tua loja.
O Pixel só é carregado na tua loja pública — nunca no painel de vendedor, para não contaminares os teus dados com a tua própria navegação.

Benchmark Shopify

Análise de como a Shopify implementa as funcionalidades que a VendaJá pretende replicar e superar.

🎨 Editor de Temas
A Shopify usa um sistema de secções onde cada bloco é um componente independente editável. O editor sincroniza via postMessage entre o iframe da loja e o painel de administração. As alterações são representadas num esquema JSON que define a ordem e configuração de cada secção.
📦 Gestão de Produtos
O formulário é dividido em painéis colapsáveis. As imagens são carregadas via upload directo para CDN com processamento assíncrono. As variantes são geradas através de uma matriz de opções — ao definir "Cor: Azul, Verde" e "Tamanho: S, M", o sistema gera automaticamente todas as combinações possíveis.
⚡ Limitações da Shopify — oportunidades para a VendaJá
  • Não foi desenhada para mercados com COD como modelo principal
  • Sem integração nativa com M-Pesa ou E-Mola
  • Suporte a mercados africanos superficial e sem localização real
  • Interface em inglês por defeito — localização limitada e cara
  • Sem catálogo de fornecedores locais integrado
  • Sem modo offline para mercados com conectividade instável

Especificação Funcional

Regras de negócio, limites por plano e comportamentos esperados de cada funcionalidade principal.

2.1 Editor de Temas

Regras de Negócio
  • Alterações ficam em rascunho até o utilizador clicar em "Publicar"
  • Rascunho guardado automaticamente a cada 30 segundos
  • Possível reverter para a última versão publicada a qualquer momento
  • Cada tema tem um esquema JSON que define quais secções são editáveis
  • As secções têm ordem, visibilidade e configurações individuais
  • Temas Refresh e Prestige bloqueados para utilizadores no Plano Básico

Arquitectura Backend

Stack de servidor e serviços independentes por domínio funcional.

Runtime
Node.js com Fastify — mais rápido que Express, TypeScript nativo, validação integrada.
Base de Dados
PostgreSQL para dados relacionais + Redis para cache de sessões e filas.
Armazenamento
Cloudflare R2 — compatível com S3, sem custos de saída. Ideal para imagens.
Filas
BullMQ + Redis para processamento de imagens e notificações WhatsApp.
Serviços principais
API Gateway (Fastify)
    ├── Auth Service         → registo, login, tokens, planos
    ├── Store Service        → lojas, temas, configurações
    ├── Product Service      → produtos, variantes, inventário
    ├── Order Service        → pedidos COD, estados
    ├── Theme Service        → secções, rascunho, publicação
    ├── Media Service        → upload, Sharp.js, CDN R2
    ├── Subscription Service → planos, M-Pesa/E-Mola
    └── Notification Service → WhatsApp, email

Base de Dados

Schema PostgreSQL com tabelas principais, tipos e relações.

SQL — Schema PostgreSQL
CREATE TABLE users (
  id               UUID PRIMARY KEY,
  nome             VARCHAR(255) NOT NULL,
  email            VARCHAR(255) UNIQUE NOT NULL,
  telefone         VARCHAR(20),
  password_hash    VARCHAR(255) NOT NULL,
  plano            ENUM('basico', 'pro', 'premium'),
  plano_valido_ate TIMESTAMP,
  criado_em        TIMESTAMP DEFAULT NOW()
);

CREATE TABLE stores (
  id            UUID PRIMARY KEY,
  user_id       UUID REFERENCES users(id),
  nome          VARCHAR(255) NOT NULL,
  slug          VARCHAR(255) UNIQUE NOT NULL,
  tema_activo   VARCHAR(50) DEFAULT 'spotlight',
  configuracoes JSONB,
  publicado     BOOLEAN DEFAULT false
);

CREATE TABLE products (
  id       UUID PRIMARY KEY,
  store_id UUID REFERENCES stores(id),
  nome     VARCHAR(255) NOT NULL,
  preco    DECIMAL(10,2) NOT NULL,
  custo    DECIMAL(10,2),
  estado   ENUM('activo', 'rascunho', 'arquivado'),
  seo_url  VARCHAR(255)
);

CREATE TABLE orders (
  id           UUID PRIMARY KEY,
  store_id     UUID REFERENCES stores(id),
  numero       VARCHAR(20) UNIQUE NOT NULL,
  cliente_nome VARCHAR(255) NOT NULL,
  cliente_tel  VARCHAR(20) NOT NULL,
  total        DECIMAL(10,2) NOT NULL,
  estado       ENUM('pendente','em_transito','entregue','cancelado')
);

CREATE TABLE subscriptions (
  id      UUID PRIMARY KEY,
  user_id UUID REFERENCES users(id),
  plano   ENUM('basico', 'pro', 'premium'),
  valor   DECIMAL(10,2),
  metodo  ENUM('mpesa', 'emola'),
  estado  ENUM('activo', 'expirado', 'cancelado'),
  fim     TIMESTAMP
);

Diferenciais Competitivos

O que a VendaJá pode fazer que a Shopify, WooCommerce e Nuvemshop não fazem para Moçambique.

DIFERENCIAL 01
COD como modelo central, não como opção
Na Shopify o pagamento na entrega é uma configuração escondida. Na VendaJá é o modelo padrão — estados de pedido, confirmações e transferências via M-Pesa construídos à volta do COD.
DIFERENCIAL 02
Integração nativa M-Pesa e E-Mola
Cobrança automática de mensalidade, recebimento COD e relatórios em meticais. Sem conversão de moeda, sem intermediários estrangeiros.
DIFERENCIAL 03
Editor simplificado para não-técnicos
Assistente guiado: "Escolhe a cor → Carrega o logo → Adiciona produtos → Publica". Sem terminologia técnica, sem configuração complexa.
DIFERENCIAL 04
WhatsApp integrado no fluxo operacional
Notificações de pedidos directamente no WhatsApp do vendedor. Partilha da loja com um toque. WhatsApp é a plataforma dominante em Moçambique.
DIFERENCIAL 05
Catálogo de fornecedores moçambicanos
Marketplace B2B dentro da plataforma onde fornecedores locais listam produtos e dropshippers importam directamente — sem depender do AliExpress.
DIFERENCIAL 06
Modo offline parcial
Pedidos guardados localmente (IndexedDB + Service Workers) e sincronizados quando a ligação voltar. Em Moçambique a conectividade é instável.
